WorkFlow (GitFlow)
===============================
Наименование веток
----------------
*master*- главная ветка (по умолчанию), рабочий билд
*develop* - ветка разработки, для слияния различных фичей, мержится в мастер, когда появляется рабочая версия
*feature/{номер связанной карточки}-{краткое описание функциональности}* - функциональная ветка, для приращения какой-либо функциональности.
**Пример названия:** feature/22-add-registration
*fix/{номер связанной карточки (если есть)}-{краткое описание исправления}* - ветка для исправлений
**Пример названия:** fix/delete-unneccessary-fields
*Чтобы видеть номера карточек, используйте расширение из вложения или аналоги*
Краткая инструкция
----------------
1. Склонировать проект
2. Создать свою ветку от ветки develop
3. После окончания работы, запушить результат
4. Создать **Merge Request** (ссылку добавьте в тикет) в ветку develop
5. Все разработчики проекта должны оставить лайк или Approve (роль Developer).
6. PM должен замержить запрос.
